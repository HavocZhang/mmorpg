连接生命周期数据流

客户端 TCP 连接 → tcp_listener: IP黑名单检查 + 连接限流 → handshake: 读取16B包头 → 校验版本 → 读取包体 → CRC32校验 → AES-GCM解密 → JSON解析 → validate_token: 非空 + 长度≥8 + 10分钟TTL + 60秒时钟偏差 → session_mgr.create_session: 顶号旧会话 → 雪花ID → 创建Session → 返回(session_id, send_rx) → WriteLoop(write_half, send_rx): 16ms合包 + 三级优先级 + 拥堵降级 → ReadLoop(read_half, decoder): 解码包 → 限流检查 → 上行路由到逻辑服 → 任一循环结束 → remove_session → 连接关闭

Redis 集群设计

Key 设计

gate:node:{node_id} → JSON(node_info) TTL 30s // 节点信息 gate:nodes → SET{node_id, ...} // 节点集合 gate:heartbeat:{node_id} → timestamp TTL 10s // 心跳 gate:route:{uid} → gate_node_id TTL 1h // 玩家路由 gate:msg:{node_id} → PubSub channel // 跨网关消息

降级策略

  • Redis 不可用时，网关仍可正常运行（仅本地能力）
  • 所有 Redis 操作失败时记录 warn 日志但不阻断主流程
  • PubSub 断线自动重连（3秒间隔）

优雅停机流程

1. 收到 SIGTERM/SIGINT 信号 2. 停止 TCP 接受器（abort tcp_handle） 3. 等待 5 秒 grace period（让存量消息处理完） 4. 清理所有在线会话（kick_session "网关停机"） 5. 注销集群节点（DEL gate:node:{id} + SREM gate:nodes） 6. 停止 HTTP 监控服务（abort admin_handle）
